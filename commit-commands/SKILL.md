# Commit Commands

This skill adapts Anthropic's `commit-commands` Claude plugin for Codex. The original command prompts are kept in `commands/`; use them as reference material, then follow Codex's local Git safety rules and the user's current request.

## Core Workflows

### Commit changes

Use for requests like "commit this", "git一下", "save latest changes", or "create a commit".

1. Inspect repository state with `git status`, `git diff`, and `git diff --cached`.
2. Inspect recent commit style with `git log --oneline -10`.
3. Check for sensitive files before staging, especially `.env`, logs, credentials, cookies, tokens, local databases, caches, and generated binaries.
4. Stage only files that belong to the user's requested scope.
5. Create one focused commit with a clear message that matches the repository's style.
6. Report the commit hash and final `git status`.

### Commit and push

Use when the user asks to commit and push to the configured remote.

1. Complete the commit workflow.
2. Verify the current branch and remote with `git branch --show-current` and `git remote -v`.
3. Push with the repository's normal upstream configuration.
4. Report the branch, remote, commit hash, and final status.

### Commit, push, and open a PR

Use only when the user explicitly asks for a pull request.

1. Complete the commit workflow.
2. If currently on the default branch and the repository convention expects feature branches, create a new branch before committing.
3. Push the branch to `origin`.
4. Use GitHub CLI `gh pr create` only when it is installed, authenticated, and the remote is a GitHub repository.
5. Include a concise PR summary and test plan.
6. Report the PR URL.

### Clean gone branches

Use when the user asks to remove local branches whose upstream branches were deleted.

1. Run `git fetch --prune` when appropriate.
2. Inspect branches with `git branch -v`.
3. Inspect worktrees with `git worktree list`.
4. For branches marked `[gone]`, remove associated worktrees before deleting branches.
5. Never delete the current branch.
6. Report removed branches and any skipped branches.

## Safety Rules

- Never run destructive Git commands such as `git reset --hard` or `git checkout --` unless the user explicitly requests that exact operation.
- Never include secrets, credentials, cookies, tokens, private keys, local account profiles, caches, logs, databases, or generated bytecode in commits.
- Do not rewrite history, force push, or delete branches unless the user explicitly asks.
- Preserve unrelated user changes. If the worktree contains unrelated modifications, stage only the requested files or explain what will be included before committing.
- If a push or PR operation needs authentication that is not available, stop and report the blocker.
