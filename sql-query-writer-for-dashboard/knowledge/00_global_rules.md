# 全局 SQL 生成规则

## 1. 查询引擎

只使用 Presto。

## 2. 表名规则

所有 SQL 必须使用完整库名.表名。

## 3. 分区规则

分区表必须加 dt。
小时表建议加 dt 和 hour。

## 4. 探索查询规则

简单查询、抽样查询、字段探索查询必须加 limit。

## 5. department_name 范围限定规则

凡字段名中包含 department_name，或业务语义为部门/业务线/学部/小组/架构，均必须加过滤条件。

如果用户未提供具体部门取值，使用占位符：

```text
'<一级部门名称>'
'<二级部门名称>'
'<三级部门名称>'
```

并在输出解释中提示用户替换。

## 6. 时间口径规则

默认时间口径不固定，必须由用户需求决定。
如果需求未说明时间范围，则不得擅自扩大扫描范围。
探索型查询可使用单日单小时模板。

### 模板取数时间参数规则

当 SQL 用于平台“模板取数”并需要维护日期/时间区间参数时，必须使用和过滤列同名的半开区间参数：

```sql
字段名 >= ${字段名:1}
and 字段名 < ${字段名:2}
```

示例：

```sql
dt >= ${dt:1}
and dt < ${dt:2}

trade_time >= ${trade_time:1}
and trade_time < ${trade_time:2}
```

不得在模板参数外层添加 `cast()`、`date()`、`substr()` 等函数，也不得使用 `begin_xxx` / `end_xxx` 作为参数名。如果需要用实际日期验证 SQL，可单独生成验证版 SQL，将模板参数替换为带引号的实际字面量；模板版本仍必须保留无 `cast` 的参数格式。

## 7. SQL 输出规则

必须输出 SQL + 解释。
解释必须包括：

- 使用表；
- 字段；
- 过滤条件；
- 分区条件；
- 范围限定；
- 指标口径；
- 待确认项。

## 8. 临时表 grade 通配约定

- `temp_table.dingxi01_cost.grade = '0'` 表示全年级通配（该渠道成本/目标适用于所有年级）。生成 LEFT JOIN 时必须使用 `(ct.grade = zz.grade_1 or ct.grade = '0')`，否则仅 `grade = '0'` 的行无法匹配任何具体年级值。
- `temp_table.dingxi01_daoke_1_6_t` 无 grade 通配约定，`grade` 必须精确匹配。

## 9. 数据透视表比值指标规则

当 SQL 结果用于看板数据透视表展示，且指标为比值、率、占比、单效、人均、客单价等需要分子和分母的计算时，默认输出英文分子字段和英文分母字段，让看板用 `sum(分子) / sum(分母)` 重算；不要只输出已计算好的行级比值。行级比值可作为单行参考字段，但必须标注不适合在透视表中 `sum` 或 `avg`。
