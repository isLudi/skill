# 全局 SQL 生成规则

## 1. 查询引擎

只使用 Presto。

## 2. 表名规则

所有 SQL 必须使用完整库名.表名。

## 3. 分区规则

分区表必须加 dt。
小时表建议加 dt 和 hour。

## 4. 探索查询规则

简单查询、抽样查询、字段探索查询必须加 limit。

## 5. department_name 范围限定规则

凡字段名中包含 department_name，或业务语义为部门/业务线/学部/小组/架构，均必须加过滤条件。

如果用户未提供具体部门取值，使用占位符：

```text
'<一级部门名称>'
'<二级部门名称>'
'<三级部门名称>'
```

并在输出解释中提示用户替换。

## 6. 时间口径规则

默认时间口径不固定，必须由用户需求决定。
如果需求未说明时间范围，则不得擅自扩大扫描范围。
探索型查询可使用单日单小时模板。

## 7. SQL 输出规则

必须输出 SQL + 解释。
解释必须包括：

- 使用表；
- 字段；
- 过滤条件；
- 分区条件；
- 范围限定；
- 指标口径；
- 待确认项。
