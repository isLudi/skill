# 市场顾问正价课订单明细（财务归属口径）模板

## 1. 定位

- 业务域：`market_consultant`
- 模板编号：M2
- 用途：按支付期次导出正向支付的正价课订单明细，并同时保留财务金额、线索归因、运营转化和去重标记。
- 精品班学部版：`resources/raw_sql/market_positive_order_detail_finance_jpb_20260718.sql`
- 市场部版：`resources/raw_sql/market_positive_order_detail_finance_market_20260718.sql`

两份 SQL 是同一母逻辑的范围变体。当前保留两份已执行版本，避免只替换一个部门谓词后造成线索侧、财务侧和运营侧范围不一致。

## 2. QuerySpec 骨架

| 槽位 | 取值/要求 |
|---|---|
| `domain` | `market_consultant` |
| `intent` | 财务支付期次正价课订单明细 |
| `time_range` | 明确支付期次列表或半开期次范围 |
| `business_scope` | 选择“精品班学部版”或“市场部/市场顾问版” |
| `calculation_grain` | 底层流水 + 订单归因明细；中间层按订单/父订单窗口去重 |
| `output_grain` | 订单—班级明细行，附订单级和父订单级去重字段 |
| `candidate_tables` | 线索全链路、财务业绩扩展、service 订单归因明细 |
| `join_path` | 订单/父订单与 `lead_id` 归因，多阶段人工 SQL |

## 3. 版本选择

| 需求 | 使用 raw SQL | 核心运营范围标记 |
|---|---|---|
| 精品班学部正价课订单 | `market_positive_order_detail_finance_jpb_20260718.sql` | `is_lead_jpb_scope` |
| 市场部/市场顾问正价课订单 | `market_positive_order_detail_finance_market_20260718.sql` | `is_lead_market_scope` |

不要把两份 SQL 直接 union 后再汇总；范围可能相交，必须先定义跨学部去重键和归属优先级。

## 4. 输出字段组

- 期次链：支付期次、交易流水期次、线索期次、订单归因流水期次、跨期类型。
- 渠道与组织：渠道、规则、顾问、主管、经理及多级部门。
- 标识：`lead_id`、用户、订单、父订单、班级、课程。
- 金额：实付、收入、退款、净流水、运营归因金额。
- 去重：订单行号、父订单行号、订单数/金额去重字段。
- 诊断：正向支付底层行数、父订单全部流水行数、订单归因底层行数。

## 5. 调用时必须替换的参数

1. 支付期次列表；历史样例为 `20260619期`、`20260626期`、`20260703期`、`20260710期`。
2. 业务范围版本；切换范围时必须同步核对线索侧范围、财务业绩范围、运营口径标记和最终去重条件。
3. 最新 `dt/hour` 快照偏移；三张小时表要使用相容的快照。
4. 最终输出是否保留订单—班级粒度，或另建订单级汇总。

## 6. 去重与汇总规则

- 订单数使用 `order_row_number = 1` 的去重字段。
- 实付金额使用 `实付金额_去重`，不要对每个班级明细行的原始金额直接求和。
- 父订单净流水与退款使用 `parent_order_row_number = 1` 的字段。
- 运营口径订单数还要求范围标记、转化订单标记以及订单归因期次与支付期次一致。

## 7. 已验证样例

- 精品班学部：55,569 个明细行、55,564 个去重正价课订单、41,062 个运营口径匹配；去重实付金额 113,775,845.24 元。
- 市场部：8,145 个明细行/去重订单、6,806 个运营口径匹配；去重实付金额 14,395,355.45 元。

## 8. 风险与门禁

- `paid_period_name`、`lead_id` 等字段来自已成功执行的历史 SQL；若共享物理目录尚未登记当前字段，必须先经数据地图同步或有界 probe 复核，不能手工补市场表目录。
- 多表、多阶段窗口和归因逻辑不进入自动编译；QueryPlan 必须标记人工 SQL。
- 明细行和去重字段混用会造成订单数、金额和退款重复。
