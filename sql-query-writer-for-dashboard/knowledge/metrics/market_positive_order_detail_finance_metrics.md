# 市场顾问正价课订单明细财务口径字段

## 1. 适用 SQL

- `resources/raw_sql/market_positive_order_detail_finance_jpb_20260718.sql`
- `resources/raw_sql/market_positive_order_detail_finance_market_20260718.sql`

## 2. 指标粒度

SQL 输出为订单—班级明细，订单数和金额汇总必须使用对应去重字段。

## 3. 关键指标

| 展示字段 | 计算口径 | 聚合规则 |
|---|---|---|
| `正价课订单数_去重` | `order_row_number = 1` 时计 1 | 可 sum |
| `实付金额_去重` | `order_row_number = 1` 时输出 `gross_paid_yuan` | 可 sum |
| `父订单净流水_去重` | `parent_order_row_number = 1` 时输出父订单净流水 | 可 sum |
| `父订单退款额_去重` | `parent_order_row_number = 1` 时输出父订单退款 | 可 sum |
| `运营口径订单数_去重` | 订单首行 + 转化订单 + 目标范围 + 归因期次等于支付期次 | 可 sum；不能替代财务正价课订单数 |
| `正向支付底层行数` | 正向支付源行计数 | 诊断字段，不作为订单数 |
| `父订单全部流水行数` | 父订单全部交易事件行数 | 诊断字段 |
| `订单归因底层行数` | service 归因明细行数 | 诊断字段 |

## 4. 两套订单数不可混用

- 财务正价课订单数回答“正向支付订单有多少”。
- 运营口径订单数回答“同时满足线索范围、转化标记和当期归因的订单有多少”。
- 两者差值用于归因覆盖诊断，不应在未说明口径时互相替代。
