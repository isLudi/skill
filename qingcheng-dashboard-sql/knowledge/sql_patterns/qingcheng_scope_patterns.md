# 青橙范围限定模式

## 1. 单字段范围

```sql
and t.<department_field> = '青橙项目部'
```

仅当已确认该字段的青橙取值就是 `青橙项目部` 时使用。

## 2. 多层级范围

```sql
and t.<first_level_department_field> = '<青橙一级部门名称>'
and t.<second_level_department_field> = '<青橙二级部门名称>'
and t.<third_level_department_field> = '<青橙三级部门/项目名称>'
```

层级字段未确认时，不得编造层级名称。

## 3. 课程部门与员工部门

同一张表同时存在课程部门和员工部门时，必须根据指标含义选择范围字段：

- 分析业绩归属或员工动作，优先确认员工/业绩部门字段；
- 分析课程、商品、学科或产品，优先确认课程/商品部门字段；
- 两者同时影响口径时，必须同时说明。

## 4. 跨域污染检查

如果 SQL 中出现 `市场顾问部`、评优、参评名单、人产或市场顾问专属临时表，不得默认作为青橙范围。应先询问用户或标注“待人工确认”。
