# 全局 SQL 生成规则

## 1. 查询引擎

只使用 Presto。

## 2. 表名规则

所有物理表必须使用完整 `库名.表名`。

## 3. 分区规则

分区表必须加 `dt`。

小时表建议同时加 `dt` 和 `hour`。如果使用最新小时，必须说明最新小时的选择逻辑。

## 4. 探索查询规则

简单查询、抽样查询、字段探索查询必须加 `limit`。

## 5. 青橙范围限定规则

凡字段名中包含 `department_name`，或业务语义为部门、业务线、学部、项目、小组、团队、员工架构、虚拟架构时，均必须加过滤条件。

如果用户未提供具体青橙取值，使用占位符：

```text
'<青橙一级部门名称>'
'<青橙二级部门名称>'
'<青橙项目/团队名称>'
```

并在输出解释中提示用户替换后执行。

## 6. 时间口径规则

默认时间口径不固定，必须由用户需求决定。

如果需求未说明时间范围，不得擅自扩大扫描范围。探索型查询可使用单日单小时模板。

## 7. 跨域隔离规则

不得默认套用市场顾问部、市场顾问评优、参评名单、人产、市场顾问专属渠道 CASE 或市场顾问临时表口径。

如果用户提供的青橙历史 SQL 中包含疑似其他部门逻辑，先标记“待人工确认”，不要直接沉淀为青橙标准口径。

## 8. 知识库写入控制规则

**禁止直接写入知识库。** 所有对知识库的新增、修改、删除操作，必须先经过用户确认审核。

具体包括：

- 禁止直接新增或修改 `resources/raw_sql/` 下的 SQL 文件
- 禁止直接新增或修改 `knowledge/tables/`、`knowledge/temp_tables/`、`knowledge/metrics/`、`knowledge/dashboards/`、`knowledge/joins/`、`knowledge/sql_patterns/` 下的知识文档
- 禁止直接更新 `knowledge/01_table_index.md`、`knowledge/update_log/changelog.md`
- 禁止直接运行 `scripts/ingest_dashboard_sql.py` 做自动入库

**正确流程：**

1. 生成 SQL 或知识文档后，先输出给用户审阅
2. 用户明确确认（如"可以入库"、"写入知识库"）后，再执行写入
3. 临时 SQL 文件可写入 `C:\Users\Ludim\.codex\runtime\` 目录

## 9. SQL 输出规则

必须输出 SQL + 解释。解释必须包括：

- 查询目的；
- 使用表；
- 使用临时表；
- 字段；
- 过滤条件；
- 分区条件；
- 青橙范围限定；
- 指标口径；
- join 关系；
- 是否加 `limit`；
- 待确认项。
